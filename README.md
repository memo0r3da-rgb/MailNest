# MailNest Temporary Email

MailNest موقع بريد مؤقت عربي جاهز للنشر كنسخة static، مع صفحات محتوى وثقة، أماكن إعلانات، ولوحة تحكم محلية لإضافة صفحات ومقالات وسكشنز.

## التشغيل المحلي

```powershell
python -m http.server 4173
```

ثم افتح:

```text
http://127.0.0.1:4173
```

## النشر

المشروع مناسب للنشر على Cloudflare Pages أو GitHub Pages أو Netlify أو Vercel. النسخة الحالية تعمل بدون باك إند، وتعتمد على ملفات static فقط.

ملفات Cloudflare المهمة:

- `_headers`: إعدادات أمان أساسية.
- `_redirects`: تحويلات روابط نظيفة مثل `/about` و`/blog`.
- `robots.txt` و`sitemap.xml`: للفهرسة.
- `ads.txt`: لتعريف حساب Google AdSense.

## الملفات المهمة

- `index.html`: الصفحة الرئيسية وأداة البريد المؤقت.
- `blog.html` و`article.html`: المقالات.
- `about.html`, `privacy.html`, `terms.html`, `faq.html`, `contact.html`: صفحات الثقة اللازمة للمستخدمين وAdSense.
- `admin.html`: لوحة التحكم المخفية، غير مرتبطة من الصفحة الرئيسية.
- `site.js`, `mail.js`, `admin.js`: منطق الموقع ولوحة التحكم.
- `site-data.js`, `seo-articles.js`: بيانات الصفحات والمقالات الافتراضية.
- `assets/mailnest-logo.png`: صورة لوجو الموقع.

## لوحة التحكم

افتح:

```text
http://127.0.0.1:4173/admin.html
```

بيانات الدخول التجريبية:

```text
admin@mailnest.local
MailNest@2026
```

لوحة التحكم الحالية تحفظ المحتوى في `localStorage` داخل نفس المتصفح. هذا مناسب كبداية static، لكن عند تحويل الموقع لمنتج أكبر يفضل إضافة باك إند وقاعدة بيانات وتسجيل دخول آمن.

## AdSense

تم تجهيز الموقع بصفحات الخصوصية والشروط والتواصل والمقالات والتنقل الواضح ومساحات الإعلانات. قبل المراجعة النهائية:

1. تأكد أن `ads.txt` يحتوي Publisher ID الصحيح.
2. تأكد أن كود AdSense موجود داخل صفحات الموقع.
3. أضف محتوى أصلي أكثر مع الوقت.
4. راقب Search Console إلى أن يتم قبول `sitemap.xml` وفهرسة الصفحات.

## التواصل

نموذج التواصل في `contact.html` يرسل إلى:

```text
memo0r3da@gmail.com
```

النموذج يستخدم FormSubmit، وقد تحتاج أول رسالة إلى تأكيد من Gmail لتفعيل استقبال الرسائل.
